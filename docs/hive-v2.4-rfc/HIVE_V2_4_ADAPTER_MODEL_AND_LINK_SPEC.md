# Agent Hive v2.4 Adapter Model and Hive Link Spec

Status: Proposed  
Date: 2026-03-27

## 1. Purpose

This document freezes the technical contract for v2.4 integrations.

It covers:

- adapter families
- integration levels
- governance modes
- Hive Link
- normalized trajectories
- capability reporting
- artifact requirements

## 2. Adapter families

### 2.1 WorkerSessionAdapter

Use for bounded work sessions with a clear lifecycle.

Required operations:

- `probe()`
- `prepare()`
- `open_session()`
- `attach_session()`
- `stream_events()`
- `send_steer()`
- `collect_artifacts()`
- `close_session()`

Expected primary user story:
> “I am working in a coding session. Hive should supervise this run.”

### 2.2 DelegateGatewayAdapter

Use for long-lived gateway/session platforms.

Required operations:

- `probe()`
- `list_sessions()`
- `attach_delegate_session()`
- `stream_events()`
- `send_steer()`
- `publish_note()`
- `collect_artifacts()`
- `detach_delegate_session()`

Expected primary user story:
> “I already have an agent/gateway running. Hive should observe and steer it.”

## 3. Integration levels

### 3.1 Enum

```text
pack
companion
attach
managed
```

### 3.2 Semantics

#### pack
Installable/discoverable in the native ecosystem. No session binding required.

#### companion
Provides native actions inside the harness. Does not require attach or managed mode.

#### attach
Binds an existing harness session to Hive and streams normalized events.

#### managed
Hive launches and governs the session.

## 4. Governance modes

### 4.1 Enum

```text
advisory
governed
```

### 4.2 Semantics

#### advisory
Hive may:
- observe
- search
- suggest work
- accept notes and steering
- record artifacts

Hive does **not** own:
- outer sandbox
- launch context
- session policy enforcement

#### governed
Hive owns:
- runpack
- task/run linkage
- worktree
- policy
- outer sandbox
- promotion/evaluator flow

## 5. Capability truth model

Every integration and every attached/managed session must persist:

```json
{
  "declared": {},
  "probed": {},
  "effective": {},
  "evidence": {},
  "governance_mode": "advisory|governed",
  "integration_level": "pack|companion|attach|managed"
}
```

### 5.1 Minimum capability fields

```json
{
  "adapter_family": "worker_session|delegate_gateway",
  "launch_mode": "none|sdk|rpc|gateway_bridge|cli",
  "attach_supported": true,
  "managed_supported": false,
  "event_stream": "none|text|structured",
  "steering": "none|queued|interactive",
  "approvals": "none|native|hive-mediated",
  "artifacts": ["transcript", "diff", "logs", "session-history"],
  "context_projection": "none|virtual|filesystem|prompt-injection",
  "outer_sandbox_owned_by_hive": false
}
```

### 5.2 Truth rules

1. `declared` = what the package knows how to do in principle.
2. `probed` = what the current installation exposes right now.
3. `effective` = what the current session is actually using.
4. The console and JSON APIs show **effective** first, not declared.
5. Managed mode may only claim `outer_sandbox_owned_by_hive=true` when Hive really owns it.

## 6. Hive Link

Hive Link is the native long-lived session bridge.

### 6.1 Why it exists

Neither MCP nor ACP is sufficient for:
- run/session binding
- governance truth
- steering state
- artifact publishing
- normalized trajectories
- approval forwarding
- future backtesting inputs

Hive Link solves that.

### 6.2 Transports

Two transports are required:

- `--stdio` for local subprocess / embedded helper use
- `--ws` for gateway or remote bridge use

### 6.3 Authentication

Minimum v2.4 contract:

- local token generated by Hive integration setup
- token scoped to workspace
- token stored outside model-visible context
- token never emitted into normalized trajectories

## 7. Normalized trajectory format

Every attached or managed session must export:

```text
.hive/runs/<run-id>/trajectory.jsonl
```

For delegate sessions that are not mapped 1:1 to a run, use:

```text
.hive/delegates/<delegate-session-id>/trajectory.jsonl
```

### 7.1 Common event envelope

```json
{
  "seq": 1,
  "ts": "2026-03-27T12:34:56Z",
  "harness": "pi|openclaw|hermes",
  "adapter_family": "worker_session|delegate_gateway",
  "native_session_ref": "string",
  "run_id": "optional",
  "delegate_session_id": "optional",
  "project_id": "optional",
  "task_id": "optional",
  "campaign_id": "optional",
  "kind": "session_start",
  "payload": {},
  "raw_ref": "optional raw transcript/history pointer"
}
```

### 7.2 Required event kinds

- `session_start`
- `session_end`
- `turn_start`
- `turn_end`
- `assistant_delta`
- `user_message`
- `tool_call_start`
- `tool_call_update`
- `tool_call_end`
- `approval_request`
- `approval_decision`
- `steering_received`
- `artifact_written`
- `compaction`
- `error`

### 7.3 Optional event kinds

- `context_loaded`
- `skill_loaded`
- `subagent_spawned`
- `memory_mutation`
- `sandbox_notice`

## 8. Companion action vocabulary

To reduce cognitive load across ecosystems, every companion package should expose the same core action set, even if the native UI names differ.

Required action intents:

- `hive_next`
- `hive_search`
- `hive_open`
- `hive_attach`
- `hive_finish`
- `hive_note`
- `hive_status`

These actions should be implemented as thin wrappers over stable Hive JSON CLI and/or Hive Link.

## 10. CLI additions

### 10.1 New commands

```text
hive integrate list
hive integrate <pi|openclaw|hermes>
hive integrate doctor <pi|openclaw|hermes> --json
hive link serve [--stdio|--ws 127.0.0.1:PORT]
hive attach <harness> <native-session-ref> [--task-id ... | --campaign-id ...]
hive detach <session-or-run-id>
```

### 10.2 Command semantics

#### `hive integrate <harness>`
Interactive/setup flow. Must:
- detect local/native install
- explain supported levels on this machine
- create or verify link token/config
- install or point to native package/skill instructions
- run a link test

#### `hive integrate doctor <harness> --json`
Must return:
- detected native version
- detected install path
- supported levels
- effective capabilities
- configuration problems
- next-step suggestions

#### `hive attach ...`
Manual fallback for when the user or companion package wants to bind a live native session to Hive.

## 11. Console requirements

The run/delegate detail view must expose:

- harness name
- adapter family
- integration level
- governance mode
- native session ref
- attach origin vs managed origin
- capability snapshot
- sandbox owner
- steering history
- approvals
- normalized trajectory
- raw/native artifact pointers when available

## 12. Cross-cutting security rules

1. Tokens for Hive Link must never be shown to the model.
2. Secret-bearing environment variables are excluded from trajectories and notes.
3. Attach mode may record sandbox provenance but may not claim sandbox ownership unless Hive truly owns it.
4. Harness-private memory stores are never bulk-imported automatically.
5. Native plugins with in-process execution risk (OpenClaw) must not be the required base path.

## 13. Deferred items

Not required for v2.4 GA:

- OpenClaw managed mode
- Hermes managed mode
- ACP-native editor runtime ownership
- richer cross-encoder/reranker tuning
- remote vector backend work
- sandbox parity follow-up items already deferred from v2.3

## 14. Release bar for this spec

This spec is satisfied only when:

- Pi, OpenClaw, and Hermes no longer share one generic adapter type
- companion packages/actions exist
- attach mode works without transcript copy/paste
- governance truth is explicit
- normalized trajectories are persisted and inspectable
