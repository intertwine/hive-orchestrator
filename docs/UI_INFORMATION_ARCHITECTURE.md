# UI Information Architecture

The current Agent Hive console is organized around exceptions and decisions, not folders.

## Home

Answers five questions fast:

1. What is running now?
2. What is blocked?
3. What needs me?
4. What changed since I last checked?
5. What should happen next?

## Runs

The unified run board. Filter by project, driver, health, and campaign. This is the main place to see live work across the portfolio.

## Inbox

Approvals, escalations, failed runs, blocked projects, and other items that actually need operator attention.

## Campaigns

Bounded multi-run initiatives with cadence, brief generation, and a shared goal across one or more projects.

## Projects

Project summaries, Program Doctor output, and compiled context preview.

## Search

Unified search across docs, tasks, runs, memory, recipes, campaigns, and projections.

## Run Detail

The deep-inspection view for one run:

- timeline
- capability snapshot and effective driver truth
- sandbox policy and backend metadata
- acceptance rationale
- evaluator results
- diff and changed files
- retrieval trace and inclusion reasons
- pending approvals and approval actions
- steering history and steering actions
- driver metadata and logs
