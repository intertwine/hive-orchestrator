# Observe And Steer

Use Hive like a control room, not a checklist editor.

## What to watch

- Home: what needs attention now
- Inbox: runs waiting for input or review
- Runs: current driver, health, and lineage
- Run detail: timeline, context manifest, diff, and evaluator evidence

## Typical loop

1. Check the home view.
2. Start work with `hive work`.
3. Let the harness work inside the prepared run worktree.
4. Finish with `hive finish`.
5. Steer only when Hive needs help choosing, pausing, or rerouting work.
